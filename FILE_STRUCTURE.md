## ⚙️ JavaScript Structure

### **script.js** - Main Functionality
**Functions:**
- `initMobileMenu()` - Toggle mobile menu on/off
- `smoothScroll()` - Smooth scroll for anchor links
- `highlightActiveNav()` - Show active page in navigation
- `observeElements()` - Fade-in animations on scroll

**When it Runs:** On page load (all pages)

### **search.js** - Player Search & Filter
**Key Variables:**
- `allPlayers` - Array of player objects from JSON

**Functions:**
- `loadPlayers()` - Fetch data from players.json
- `displayPlayers(players)` - Generate HTML for player cards
- `setupSearch()` - Attach search input listener
- `setupPositionFilter()` - Create filter dropdown & listener

**When it Runs:** Only on players.html page

**Data Flow:**
```
1. Page loads → loadPlayers()
2. Fetch players.json → allPlayers array
3. displayPlayers(allPlayers) → Show all players
4. User types in search → Filter → displayPlayers(filtered)
5. User selects position → Filter → displayPlayers(filtered)
```

### **compare.js** - Player Comparison
**Key Variables:**
- `allPlayers` - Array of all players
- `selectedPlayers` - Currently selected players to compare

**Functions:**
- `loadPlayers()` - Fetch player data
- `setupCompare()` - Create dropdown selectors
- `updateComparison()` - Get selected players
- `displayComparison()` - Generate HTML comparison table

**When it Runs:** Only on compare.html page

**Data Flow:**
```
1. Page loads → loadPlayers()
2. Populate 3 dropdowns with player options
3. User selects player → updateComparison()
4. Generate comparison table → displayComparison()
5. User changes selection → Table updates automatically
```

## 🚀 How to Extend

### Add a New Player:
1. Edit `assets/data/players.json`
2. Add new player object
3. Done! Appears in Players page automatically

### Add a New Page:
1. Create `newpage.html`
2. Include standard header/footer (copy from existing page)
3. Link in navigation on all pages
4. Style with existing `css/style.css`

### Customize Colors:
1. Edit CSS variables at top of `css/style.css`
2. Change `--primary-blue`, `--secondary-red`, etc.
3. All colors update automatically site-wide

### Add More Features:
1. Create new `.js` file in `js/` folder
2. Load it with `<script>` tag on needed pages
3. Follow same naming convention

## 📌 Key Design Decisions

1. **No Framework** → Faster loading, simpler code
2. **JSON Data** → Easy to modify, scalable
3. **CSS Grid/Flexbox** → Responsive without extra tools
4. **Vanilla JS** → No dependencies to manage
5. **Semantic HTML** → Better SEO and accessibility
6. **Mobile-First CSS** → Better mobile experience
7. **Modular JS** → Separate files by feature
8. **Consistent Styling** → CSS variables for theming
